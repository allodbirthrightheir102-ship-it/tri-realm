SOVERAIN‑STRUCTURA — DIRECTORY STANDARD  
Registry‑Bound Institution  
Anchor: Sykesville, Maryland  

SUBJECT:
Canonical directory and repository classification for the SOVERAIN‑STRUCTURA
identity architecture. Defines the purpose, boundaries, and placement rules for
all realms, layers, and deployment repositories.

PURPOSE:
To establish a permanent, non‑ambiguous structural standard governing the
placement, classification, and function of all repositories and directories
within the SOVERAIN‑STRUCTURA system.

----------------------------------------------------------------------
V. DIRECTORY RULES & BOUNDARIES
----------------------------------------------------------------------

1. No realm may contain another realm.  
2. No pointer file may appear outside its designated realm.  
3. The MASTER INSTALLATION FILE may only exist in the INSTALLMENTS repository.  
4. TRI‑REALM may reference all realms but may not contain realm‑specific files.  
5. PUBLIC and BUNDLE layers may not contain identity or specification documents.  
6. No repository may duplicate the MASTER INSTALLATION FILE.  
7. All directories must remain non‑executable and documentation‑grade.
